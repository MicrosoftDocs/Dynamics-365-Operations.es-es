## <a name="batch-processing-for-change-based-events"></a><span data-ttu-id="58eb4-111">Procesamiento de lotes para eventos basados en cambios</span><span class="sxs-lookup"><span data-stu-id="58eb4-111">Batch processing for change-based events</span></span>

### <a name="frequency-for-change-based-events"></a><span data-ttu-id="58eb4-116">Frecuencia para los eventos basados en cambios</span><span class="sxs-lookup"><span data-stu-id="58eb4-116">Frequency for change-based events</span></span>

<span data-ttu-id="58eb4-120">Por otra parte, un trabajo por lotes que se repite con menor frecuencia, y que se programa para ocasiones en que la carga del sistema es baja, puede ayudar a mejorar el rendimiento del sistema.</span><span class="sxs-lookup"><span data-stu-id="58eb4-120">On the other hand, a batch job that recurs less often, and that is scheduled for times when the system load is low, might help improve system performance.</span></span> <span data-ttu-id="58eb4-121">Sin embargo, una baja frecuencia para el procesamiento por lotes podría no cumplir las demandas de los usuarios para las alertas oportunas.</span><span class="sxs-lookup"><span data-stu-id="58eb4-121">However, a low frequency for batch processing might not meet the users' demands for timely alerts.</span></span>

<span data-ttu-id="58eb4-122">Por lo tanto, al configurar la frecuencia del procesamiento por lotes para eventos basados en cambios, piense en la relevancia temporal de las alertas y el rendimiento de todo el sistema.</span><span class="sxs-lookup"><span data-stu-id="58eb4-122">Therefore, when you set up the frequency of batch processing for change-based events, consider the balance between the timeliness of alerts and the performance of the whole system.</span></span> <span data-ttu-id="58eb4-123">Estas consideraciones se vuelven más relevantes a medida que disminuye el número de usuarios que crean reglas de alertas.</span><span class="sxs-lookup"><span data-stu-id="58eb4-123">These considerations become more relevant as the number of users who create alert rules increases.</span></span> <span data-ttu-id="58eb4-124">La frecuencia no afecta al número de eventos que se deben procesar.</span><span class="sxs-lookup"><span data-stu-id="58eb4-124">The frequency doesn't affect the number of events that must be processed.</span></span> <span data-ttu-id="58eb4-125">Sin embargo, si varios usuarios crean reglas, se deben realizar varias comprobaciones.</span><span class="sxs-lookup"><span data-stu-id="58eb4-125">However, if more users create rules, more checks must be done.</span></span> <span data-ttu-id="58eb4-126">Este tipo de intercambio de datos puede afectar al rendimiento del sistema.</span><span class="sxs-lookup"><span data-stu-id="58eb4-126">This type of data exchange might affect system performance.</span></span>

#### <a name="the-risks-of-low-batch-frequency"></a><span data-ttu-id="58eb4-127">Los riesgos de una frecuencia de lote baja</span><span class="sxs-lookup"><span data-stu-id="58eb4-127">The risks of low batch frequency</span></span>
<span data-ttu-id="58eb4-128">Si establece una baja frecuencia para el procesamiento por lotes para eventos basados en cambios, los datos pertinentes para las condiciones de las reglas de alerta podrían cambiarse antes de que se procese el lote.</span><span class="sxs-lookup"><span data-stu-id="58eb4-128">If you set up a low frequency for batch processing for change-based events, data that is relevant to the conditions in alert rules might be changed before the batch is processed.</span></span> <span data-ttu-id="58eb4-129">Por lo tanto, es posible que pierda alertas.</span><span class="sxs-lookup"><span data-stu-id="58eb4-129">Therefore, you might lose alerts.</span></span>

<span data-ttu-id="58eb4-130">Por ejemplo, una regla de alertas se configura para desencadenar una alerta cuando el evento es **contacto de cliente cambia** y la condición es **cliente = BB**.</span><span class="sxs-lookup"><span data-stu-id="58eb4-130">For example, an alert rule is set up to trigger an alert when the event is **customer contact changes** and the condition is **customer = BB**.</span></span> <span data-ttu-id="58eb4-131">Dicho de otra forma, cuando el contacto del cliente para el cliente BB se cambia, se registra el evento.</span><span class="sxs-lookup"><span data-stu-id="58eb4-131">In other words, when the customer contact for customer BB is changed, the event is logged.</span></span> <span data-ttu-id="58eb4-132">Sin embargo, el sistema de procesamiento por lotes se configura de modo que se produzca el procesamiento por lotes con menor frecuencia que la entrada de datos.</span><span class="sxs-lookup"><span data-stu-id="58eb4-132">However, the batch processing system is set up so that batch processing occurs less often than data entry.</span></span> <span data-ttu-id="58eb4-133">Si el nombre del cliente cambia de **BB** a **AA** antes de que se procese el evento, los datos de la base de datos ya no coincidirán con la condición de la regla, **cliente = BB**.</span><span class="sxs-lookup"><span data-stu-id="58eb4-133">If the customer name is changed from **BB** to **AA** before the event is processed, the data in the database no longer matches the condition in the rule, **customer = BB**.</span></span> <span data-ttu-id="58eb4-134">Por lo tanto, cuando el evento se procese finalmente, no se generará ninguna alerta.</span><span class="sxs-lookup"><span data-stu-id="58eb4-134">Therefore, when the event is finally processed, no alert is generated.</span></span>

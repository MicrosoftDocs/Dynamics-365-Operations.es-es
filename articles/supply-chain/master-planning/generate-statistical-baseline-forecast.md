# <a name="generate-a-statistical-baseline-forecast"></a><span data-ttu-id="5cf89-103">Generar previsión estadística de línea base</span><span class="sxs-lookup"><span data-stu-id="5cf89-103">Generate a statistical baseline forecast</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="5cf89-104">Este artículo proporciona información sobre los parámetros y los filtros que se usan en el cálculo de previsión de demanda.</span><span class="sxs-lookup"><span data-stu-id="5cf89-104">This article provides information about the parameters and filters that are used in the calculation of demand forecasting.</span></span> 

<span data-ttu-id="5cf89-105">Cuando se crea una previsión de línea base, primero debe especificar los parámetros y los filtros que se usan en el cálculo.</span><span class="sxs-lookup"><span data-stu-id="5cf89-105">When you create a baseline forecast, you must first specify the parameters and filters that are used in the calculation.</span></span> <span data-ttu-id="5cf89-106">Por ejemplo, puede crear una previsión de línea base que estime la demanda en función de los datos de transacción desde último año para una empresa específica, para el mes siguiente y para un grupo seleccionado de artículos.</span><span class="sxs-lookup"><span data-stu-id="5cf89-106">For example, you can create a baseline forecast that estimates demand based on transaction data from the past year for a specific company, for the coming month, and for a selected group of items.</span></span> 

<span data-ttu-id="5cf89-107">Para generar una previsión de demanda, vaya a **Planificación maestra &gt; Previsión &gt; Previsión de demanda &gt; Generar previsión estadística de línea base**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span> 

<span data-ttu-id="5cf89-108">El cubo de previsión se puede seleccionar en el momento de generar la previsión.</span><span class="sxs-lookup"><span data-stu-id="5cf89-108">The forecast bucket can be selected at forecast generation time.</span></span> <span data-ttu-id="5cf89-109">Los valores disponibles: Día, Semana y Mes.</span><span class="sxs-lookup"><span data-stu-id="5cf89-109">The available values are: Day, Week, and Month.</span></span> 

<span data-ttu-id="5cf89-110">El número de depósitos para los que calcular una previsión está definido en el campo **Horizonte de previsión**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span> 

<span data-ttu-id="5cf89-111">Cuando la estrategia de previsión se establece en **Copiar por encima de demanda histórica**, el final del horizonte histórico se ignora.</span><span class="sxs-lookup"><span data-stu-id="5cf89-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="5cf89-112">El sistema copia el número de depósitos especificado en el campo **Horizonte de previsión** en la demanda planificada, comenzando desde la fecha establecida en el campo **Desde la fecha** en **Horizonte histórico**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="5cf89-113">Copiando la demanda histórica a partir de una fecha concreta, los planificadores de producción pueden realizar el plan para el siguiente trimestre de dos maneras:</span><span class="sxs-lookup"><span data-stu-id="5cf89-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="5cf89-114">Copiando la demanda del mismo trimestre del año anterior.</span><span class="sxs-lookup"><span data-stu-id="5cf89-114">By copying the demand from the same quarter last year.</span></span>
-   <span data-ttu-id="5cf89-115">Copiando la demanda del mismo trimestre del trimestre anterior.</span><span class="sxs-lookup"><span data-stu-id="5cf89-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="5cf89-116">Para evitar confusiones en los planes de producción, algunos depósitos de previsión se pueden congelar.</span><span class="sxs-lookup"><span data-stu-id="5cf89-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="5cf89-117">Este número se establece en en campo **Límite de tiempo congelado**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="5cf89-118">En la página **Previsión de la demanda ajustada**, las celdas para cubos congelados están desactivadas, para dar una indicación visual de que estos valores no se deben cambiar.</span><span class="sxs-lookup"><span data-stu-id="5cf89-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span> 

<span data-ttu-id="5cf89-119">La fecha inicial para la previsión de la demanda de línea basa no tiene que ser la fecha actual o una fecha en el futuro.</span><span class="sxs-lookup"><span data-stu-id="5cf89-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="5cf89-120">Para establecer una fecha inicial, use el campo **Fecha de inicio de previsión de línea base: desde fecha**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-120">To set a different start date, use the **Baseline forecast start date - From date** field.</span></span> <span data-ttu-id="5cf89-121">Por ejemplo, en junio, los usuarios pueden generar una previsión para el año siguiente.</span><span class="sxs-lookup"><span data-stu-id="5cf89-121">For example, in June, users can generate a forecast for the next year.</span></span> <span data-ttu-id="5cf89-122">Dado que faltan los cubos de previsión entre el final de la demanda histórica y el inicio de la línea base, puede que las predicciones no sean exactas.</span><span class="sxs-lookup"><span data-stu-id="5cf89-122">Because the forecast buckets between the end of historical demand and the start of the baseline are missing, the predictions might not be accurate.</span></span> <span data-ttu-id="5cf89-123">Si está usando el servicio de previsión de demanda de Microsoft Dynamics 365 for Finance and Operations, existen cuatro maneras en las que puede completar los espacios que faltan.</span><span class="sxs-lookup"><span data-stu-id="5cf89-123">If you are using the Microsoft Dynamics 365 for Finance and Operations Demand forecasting service, there are four ways in which you can fill in the missing gaps.</span></span> <span data-ttu-id="5cf89-124">Puede elegir el método que desee si configura el parámetro MISSING\_VALUE\_SUBSTITUTION en la página **Parámetros de previsión de demanda**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-124">You can choose the method that you want by setting the MISSING\_VALUE\_SUBSTITUTION parameter on the **Demand forecasting parameters** page.</span></span> 

<span data-ttu-id="5cf89-125">El campo **Fecha inicial de la previsión de línea base** - **Desde la fecha** tiene que estar establecido al principio de un cubo de previsión, por ejemplo, en Estados Unidos, un domingo si el cubo de previsión es la semana.</span><span class="sxs-lookup"><span data-stu-id="5cf89-125">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="5cf89-126">El sistema ajusta automáticamente el campo **Fecha inicial de la previsión de línea base** - **Desde la fecha** para que coincida con el inicio de un cubo de previsión.</span><span class="sxs-lookup"><span data-stu-id="5cf89-126">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="5cf89-127">El campo **Fecha de inicio de previsión de línea base** - **Desde la fecha** puede establecerse en una fecha del pasado.</span><span class="sxs-lookup"><span data-stu-id="5cf89-127">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="5cf89-128">Es decir, es posible generar una previsión de demanda en el pasado.</span><span class="sxs-lookup"><span data-stu-id="5cf89-128">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="5cf89-129">Esto resulta útil, ya que permite a los usuarios retocar los parámetros del servicio de previsión de modo que la previsión estadística generado en el pasado coincide con la demanda histórica real.</span><span class="sxs-lookup"><span data-stu-id="5cf89-129">This is useful, because it lets users tweak the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="5cf89-130">Los usuarios pueden continuar usando esta configuración de parámetros para generar una previsión estadística de línea base para el futuro.</span><span class="sxs-lookup"><span data-stu-id="5cf89-130">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span> 

<span data-ttu-id="5cf89-131">Los ajustes manuales realizados en iteraciones anteriores de previsión de demanda se pueden aplicar automáticamente a la nueva previsión de línea base si se activa la casilla **Transferir ajustes manuales a las previsiones de demanda**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-131">Manual adjustments made in previous demand forecasting iterations can be automatically applied to the new baseline forecast if the **Transfer manual adjustments to the demand forecas**t check box is selected.</span></span> <span data-ttu-id="5cf89-132">Si la casilla está desactivada, los ajustes manuales no se agregan a la previsión de línea base, pero tampoco se eliminan.</span><span class="sxs-lookup"><span data-stu-id="5cf89-132">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not deleted.</span></span> <span data-ttu-id="5cf89-133">Los ajustes manuales efectuados a una previsión se pueden eliminar solo en el momento de la importación de previsión, desactivando la casilla **Guardar los ajustes manuales realizados en la previsión de la demanda de línea base**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-133">Manual adjustments made to a forecast can be deleted only at forecast import time, by clearing the **Save the manual adjustments made to the baseline demand forecast** check box.</span></span> <span data-ttu-id="5cf89-134">Los ajustes manuales se guardan en el momento de la autorización.</span><span class="sxs-lookup"><span data-stu-id="5cf89-134">Manual adjustments are saved at authorization time.</span></span> <span data-ttu-id="5cf89-135">Por lo tanto, si un usuario realiza ajustes manuales a la previsión, pero no autoriza la previsión de nuevo a Finance and Operations, los cambios se perderán.</span><span class="sxs-lookup"><span data-stu-id="5cf89-135">Therefore, if a user makes manual adjustments to the forecast, but doesn’t authorize the forecast back to Finance and Operations, the changes are lost.</span></span> <span data-ttu-id="5cf89-136">Para obtener más información acerca de los ajustes manuales y cómo funcionan, consulte [Autorización de la previsión ajustada](authorize-adjusted-forecast.md).</span><span class="sxs-lookup"><span data-stu-id="5cf89-136">For more information about manual adjustments and how they work, see [Authorizing the adjusted forecast](authorize-adjusted-forecast.md).</span></span> 

<span data-ttu-id="5cf89-137">Una generación de previsión de la demanda puede tener un nombre y comentarios para ayudar a los usuarios a identificar la previsión que se ha generado.</span><span class="sxs-lookup"><span data-stu-id="5cf89-137">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="5cf89-138">Estos valores son visibles en el historial de la generación de previsión en la página **Historial estadística de la generación de previsión de línea base**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-138">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span> 

<span data-ttu-id="5cf89-139">El grupo de planificación de empresas vinculadas, se pueden aplicar las claves de asignación de artículos y otros filtros en el momento de la generación de la previsión.</span><span class="sxs-lookup"><span data-stu-id="5cf89-139">The intercompany planning group, item allocation keys, and other filters can be applied at forecast generation time.</span></span> <span data-ttu-id="5cf89-140">Esto se pueden usar para mejorar el rendimiento o para dividir los datos en piezas manejables.</span><span class="sxs-lookup"><span data-stu-id="5cf89-140">These can be used to improve performance or to split the data into manageable chunks.</span></span> <span data-ttu-id="5cf89-141">Sin embargo, tenga en cuenta que no se genera una previsión de la demanda para los miembros de ninguna clave de ninguna clave de asignación de artículos que no está asociada a un grupo de planificación de empresas vinculadas, incluso si la clave de asignación de artículos se selecciona en la consulta.</span><span class="sxs-lookup"><span data-stu-id="5cf89-141">However, note that a demand forecast is not generated for the members of any item allocation key that is not associated with an intercompany planning group, even if the item allocation key is selected in the query.</span></span> 

<span data-ttu-id="5cf89-142">**Sugerencia**: los usuarios pueden recibir a veces mensajes de error mientras se genera una previsión de la demanda, o se completa una generación de previsión sin registro de sesión.</span><span class="sxs-lookup"><span data-stu-id="5cf89-142">**Tip**: Sometimes users might receive errors while generating a demand forecast, or forecast generation is completed with no session log.</span></span> <span data-ttu-id="5cf89-143">Esto puede suceder debido a datos de excedente en la consulta usada anteriormente para la generación de previsión.</span><span class="sxs-lookup"><span data-stu-id="5cf89-143">This can happen because of leftover data in the query that was previously used for forecast generation.</span></span> <span data-ttu-id="5cf89-144">Para corregir este problema, haga clic en **Seleccionar** para abrir la página **Consulta**, haga clic en **Restaurar** y vuelva a generar la previsión de línea base.</span><span class="sxs-lookup"><span data-stu-id="5cf89-144">To fix this issue, click **Select** to open the **Query** page, click **Reset**, and then regenerate the baseline forecast.</span></span> 

<span data-ttu-id="5cf89-145">Si la previsión no se genera para un conjunto grande de artículos, sino, por ejemplo, para un artículo o una clave de asignación de artículos a la vez, para obtener un mejor rendimiento, puede activar la casilla **Usar modo de respuesta de la solicitud** en la pestaña **Planificación maestra - configuración - previsión de demanda** - **Parámetros de previsión de demanda - Aprendizaje de Azure Machine**.</span><span class="sxs-lookup"><span data-stu-id="5cf89-145">If the forecast is not generated for a big set of items, but, for example, for one item or one item allocation key at a time, then in order to get better performance, you can select the **Use request response mode** check box on the **Master planning - Setup - Demand forecasting** - **Demand forecasting parameters - Azure Machine Learning** tab.</span></span>

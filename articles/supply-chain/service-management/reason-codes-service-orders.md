# <a name="reason-codes-for-service-orders"></a>Códigos de motivo para pedidos de servicio   

[!include [banner](../includes/banner.md)]


Puede usar códigos de motivo para ayudar a explicar el estado de un pedido de servicio cuando la fase de un pedido de servicio se actualiza. Por ejemplo, si cancela un pedido de servicio, puede seleccionar un código de motivo para la cancelación.

Para ver información sobre los códigos de motivo que se usan al realizar un seguimiento del progreso del pedido de servicio, ejecute el informe Progreso del pedido de servicio. Este informe muestra todos los pedidos de servicio, independientemente de su fase, y los códigos de motivo que se especifican cuando se actualiza una etapa del pedido de servicio.

## <a name="turn-reason-codes-on-or-off"></a>Activar o desactivar los códigos de servicio

Los códigos de motivo son opcionales. Puede decidir si es obligatorio especificar un código de motivo al actualizar un pedido de servicio a una etapa de servicio específica.

1.  Haga clic en **Gestión de servicio** \> **Configuración** \> **Pedidos de servicio** \> **Etapas de servicio**.

2.  En el formulario **Etapas de servicio**, seleccione una etapa de servicio y active la casilla **Motivo** para la etapa de servicio.

3.  Cierre el formulario para guardar los cambios.

# <a name="update-standard-costs-for-a-new-manufactured-item"></a><span data-ttu-id="7df90-103">Actualizar costes estándar para un nuevo artículo fabricado</span><span class="sxs-lookup"><span data-stu-id="7df90-103">Update standard costs for a new manufactured item</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="7df90-104">Este artículo proporciona orientación para actualizar los costes estándar para un nuevo artículo fabricado.</span><span class="sxs-lookup"><span data-stu-id="7df90-104">This article provides guidance for updating standard costs for a new manufactured item.</span></span> 

<span data-ttu-id="7df90-105">En las directrices siguientes se supone que se usa el enfoque de dos versiones para actualizar costes estándar.</span><span class="sxs-lookup"><span data-stu-id="7df90-105">The following guidelines assume that you use a two-version approach to update standard costs.</span></span> <span data-ttu-id="7df90-106">En este enfoque, una versión de gestión de costes contiene los costes estándar que se definieron originalmente para el período congelado y la segunda versión de gestión de costes contiene las actualizaciones incrementales relacionadas con los nuevos artículos fabricados.</span><span class="sxs-lookup"><span data-stu-id="7df90-106">In this approach, one costing version contains the standard costs that were originally defined for the frozen period, and the second costing version contains the incremental updates that pertain to the new manufactured items.</span></span> <span data-ttu-id="7df90-107">Las actualizaciones incrementales se especifican como registros de coste en la segunda versión de gestión de costes y, finalmente, se activan.</span><span class="sxs-lookup"><span data-stu-id="7df90-107">The incremental updates are entered as cost records in the second costing version, and eventually they are enabled.</span></span> <span data-ttu-id="7df90-108">El enfoque de dos versiones requiere que defina una segunda versión de gestión de costes.</span><span class="sxs-lookup"><span data-stu-id="7df90-108">The two-version approach requires that you define a second costing version.</span></span> <span data-ttu-id="7df90-109">Esta son las instrucciones para definir esta versión de gestión de costes:</span><span class="sxs-lookup"><span data-stu-id="7df90-109">Here are the guidelines for defining this costing version:</span></span>

-   <span data-ttu-id="7df90-110">Asigne un tipo de gestión de costes de **Coste estándar**.</span><span class="sxs-lookup"><span data-stu-id="7df90-110">Assign a costing type of **Standard cost**.</span></span>
-   <span data-ttu-id="7df90-111">Asignar un identificador significativo que indica el contenido de la versión de gestión de costes, como **2016-ACTUALIZACIONES**.</span><span class="sxs-lookup"><span data-stu-id="7df90-111">Assign a significant identifier that indicates the contents of the costing version, such as **2016-UPDATES**.</span></span>
-   <span data-ttu-id="7df90-112">En el grupo de campos **Permitir tipos de precio**, asegúrese de que **Precio de coste** se establece en **Sí**.</span><span class="sxs-lookup"><span data-stu-id="7df90-112">In the **Allow price types** field group, make sure that **Cost price** is set to **Yes**.</span></span>
-   <span data-ttu-id="7df90-113">Permita que se especifiquen registros de costes para todos los sitios (es decir, deje el campo **Sitio** en blanco).</span><span class="sxs-lookup"><span data-stu-id="7df90-113">Allow cost records to be entered for all sites (that is, leave the **Site** field blank).</span></span> <span data-ttu-id="7df90-114">Si especifica un sitio, los registros de costes se pueden especificar solo para ese sitio.</span><span class="sxs-lookup"><span data-stu-id="7df90-114">If you enter a site, cost records can be entered only for that site.</span></span>
-   <span data-ttu-id="7df90-115">Usar un principio de reserva de **Activo**.</span><span class="sxs-lookup"><span data-stu-id="7df90-115">Use a fallback principle of **Active**.</span></span>

<span data-ttu-id="7df90-116">Para agregar nuevos artículos de fabricación a lo largo del período congelado, siga estos pasos:</span><span class="sxs-lookup"><span data-stu-id="7df90-116">To add new manufacturing items throughout the frozen period, follow these steps.</span></span>

1.  <span data-ttu-id="7df90-117">Utilice la página **Configuración de la versión de gestión de costes** para permitir la entrada de registros de coste en la segunda versión de gestión de costes que contenga las actualizaciones incrementales.</span><span class="sxs-lookup"><span data-stu-id="7df90-117">Use the **Costing version setup** page to enable cost records to be entered into the second costing version that contains the incremental updates.</span></span> <span data-ttu-id="7df90-118">Impida la activación de costes pendientes donde la activación se permitirá después de que los costes pendientes se hayan definido completamente y de manera precisa.</span><span class="sxs-lookup"><span data-stu-id="7df90-118">Prevent the activation of pending costs, where activation is allowed after pending costs have been completely and accurately defined.</span></span> <span data-ttu-id="7df90-119">Indique una fecha de inicio en blanco como directiva en la versión de gestión de costes y, a continuación, especifique una fecha de inicio al especificar cada registro de coste.</span><span class="sxs-lookup"><span data-stu-id="7df90-119">Indicate a blank from-date as a policy in the costing version, and then enter the from-date when you enter each cost record.</span></span> <span data-ttu-id="7df90-120">La fecha de inicio debe representar una fecha anterior a la compra o fabricación de los artículos nuevos.</span><span class="sxs-lookup"><span data-stu-id="7df90-120">The from-date should represent a date before the new items are purchased or manufactured.</span></span>
2.  <span data-ttu-id="7df90-121">Use la página **Precio de artículo** para entrar los registros de coste de los nuevos artículos comprados.</span><span class="sxs-lookup"><span data-stu-id="7df90-121">Use the **Item price** page to enter cost records for new purchased items.</span></span> <span data-ttu-id="7df90-122">Para cada uno, especifique la versión de gestión de costes que contenga las actualizaciones incrementales y use una fecha de inicio anterior a la fecha de fabricación prevista de los nuevos artículos fabricados.</span><span class="sxs-lookup"><span data-stu-id="7df90-122">For each cost record, enter the costing version that contains incremental updates, and use a from-date that comes before the expected manufacturing date for new manufactured items.</span></span>
3.  <span data-ttu-id="7df90-123">Calcule el coste de los nuevos artículos fabricados mediante el formulario **Cálculo**.</span><span class="sxs-lookup"><span data-stu-id="7df90-123">Calculate the cost of new manufactured items by using the **Calculation** page.</span></span> <span data-ttu-id="7df90-124">Abra la página **Cálculo** en la página **Mantenimiento de la versión de gestión de costes** y, a continuación, seleccione la versión de gestión de costes que contenga las actualizaciones incrementales.</span><span class="sxs-lookup"><span data-stu-id="7df90-124">Open the **Calculation** page from the **Costing version maintenance** page, and then select the costing version that contains the incremental updates.</span></span> <span data-ttu-id="7df90-125">Utilice los criterios de selección para especificar el nuevo artículo fabricado y uno de sus componentes fabricados.</span><span class="sxs-lookup"><span data-stu-id="7df90-125">Use the selection criteria to specify the new manufactured item and any one of its manufactured components.</span></span> <span data-ttu-id="7df90-126">En una estructura de productos multinivel, también es posible que sea necesario especificar cualquier artículo principal que contenga los nuevos artículos fabricados como componente.</span><span class="sxs-lookup"><span data-stu-id="7df90-126">In a multilevel product structure, you might also have to specify any parent item that contains the new manufactured items as a component.</span></span> <span data-ttu-id="7df90-127">Especifique una fecha de inicio para el cálculo de la lista de materiales (L. MAT.) que corresponda al inicio de la fabricación de los nuevos artículos fabricados.</span><span class="sxs-lookup"><span data-stu-id="7df90-127">Enter a from-date for the bill of materials (BOM) calculation that corresponds to the start of manufacturing for the new manufactured items.</span></span> <span data-ttu-id="7df90-128">La fecha de inicio debe estar dentro de las fechas vigentes de la versión L. MAT y versión de ruta del artículo.</span><span class="sxs-lookup"><span data-stu-id="7df90-128">The from-date should be in the effective dates for the item’s BOM version and route version.</span></span> <span data-ttu-id="7df90-129">**Nota:** Un registro de costes ausente podría indicar un nuevo artículo fabricado.</span><span class="sxs-lookup"><span data-stu-id="7df90-129">**Note:** A missing cost record can indicate a new manufactured item.</span></span> <span data-ttu-id="7df90-130">Los cálculos de L. MAT pueden limitarse a los artículos con costes ausentes.</span><span class="sxs-lookup"><span data-stu-id="7df90-130">BOM calculations can be limited to items that have missing costs.</span></span>
4.  <span data-ttu-id="7df90-131">Compruebe que los costes calculados de los nuevos artículos fabricados están completos y precisos.</span><span class="sxs-lookup"><span data-stu-id="7df90-131">Verify the completeness and accuracy of the calculated costs for new manufactured items.</span></span> <span data-ttu-id="7df90-132">Utilice la página **Completar** para revisar los costes calculados de todos los registros de coste de artículos y también revisar si hay algún mensaje de aviso del registro de información.</span><span class="sxs-lookup"><span data-stu-id="7df90-132">Use the **Complete** page to review the calculated costs for each item cost record, and also review any Infolog warning messages.</span></span> <span data-ttu-id="7df90-133">También puede usar la página **Cálculo** para revisar una lista de costes calculados.</span><span class="sxs-lookup"><span data-stu-id="7df90-133">Alternatively, use the **Calculation** page to review the calculated costs.</span></span>
5.  <span data-ttu-id="7df90-134">Utilice la página **Configuración de la versión de gestión de costes**  para modificar el indicador de bloqueo y permitir la activación de los registros de costes pendientes incluidos en la segunda versión de gestión de costes.</span><span class="sxs-lookup"><span data-stu-id="7df90-134">Use the **Costing version setup** page to change the blocking flag to allow activation of the pending cost records in the second costing version.</span></span>
6.  <span data-ttu-id="7df90-135">Use la página **Activar precios** (que se abre desde la página **Mantenimiento de la versión de gestión de costes**) para habilitar todos los registros de costes pendientes en la segunda versión de gestión de costes.</span><span class="sxs-lookup"><span data-stu-id="7df90-135">Use the **Activate prices** page (which you open from the **Costing version maintenance** page) to enable all pending cost records in the second costing version.</span></span> <span data-ttu-id="7df90-136">También puede habilitar los registros de costes pendientes para los artículos individuales haciendo clic en el botón **Activar** de la página **Precio de artículo**.</span><span class="sxs-lookup"><span data-stu-id="7df90-136">You can also enable the pending cost records for individual items by clicking the **Activate** button on the **Item price** page.</span></span>
7.  <span data-ttu-id="7df90-137">Use la página **Configuración de la versión de gestión de costes** para modificar los indicadores de bloqueo de la segunda versión de gestión de costes para impedir la actualización posterior de los datos.</span><span class="sxs-lookup"><span data-stu-id="7df90-137">Use the **Costing version setup** page to change the blocking flags in the second costing version to prevent additional data maintenance.</span></span> <span data-ttu-id="7df90-138">Las directivas de bloqueo impiden la entrada de nuevos costes pendientes y la activación de costes pendientes.</span><span class="sxs-lookup"><span data-stu-id="7df90-138">The blocking policies prevent the entry of new pending costs and the activation of pending costs.</span></span>

# <a name="set-up-fraud-alerts"></a><span data-ttu-id="a3d30-104">Configurar avisos de fraude</span><span class="sxs-lookup"><span data-stu-id="a3d30-104">Set up fraud alerts</span></span>

[!include[banner](includes/banner.md)]

<span data-ttu-id="a3d30-105">Este tema explica cómo configurar criterios y reglas para poner en espera pedidos de ventas potencialmente fraudulentos para revisión adicional.</span><span class="sxs-lookup"><span data-stu-id="a3d30-105">This topic explains how to set up criteria and rules to place potentially fraudulent sales orders on hold for further review.</span></span> <span data-ttu-id="a3d30-106">La funcionalidad de revisión de fraudes se usa para determinar la validez de la información en un pedido de ventas.</span><span class="sxs-lookup"><span data-stu-id="a3d30-106">Fraud review functionality is used to determine the validity of information in a sales order.</span></span> <span data-ttu-id="a3d30-107">Si la información del pedido de ventas parece para ser cuestionable en función de los criterios y las reglas contra el fraude de una organización, el pedido puede ser puesto en espera para revisión adicional a cargo de un gestor.</span><span class="sxs-lookup"><span data-stu-id="a3d30-107">If the information in the sales order appears to be questionable based on an organization’s fraud criteria and rules, the order may be put on hold for further review by an administrator.</span></span>

> [!NOTE]
> <span data-ttu-id="a3d30-108">Esta función sólo se puede usar con el procesamiento de pedidos de ventas del canal de centro de llamadas de Retail.</span><span class="sxs-lookup"><span data-stu-id="a3d30-108">This feature can only be used with Retail Call Center channel sales order processing.</span></span> 

<span data-ttu-id="a3d30-109">Cuando se define el canal de centro de llamadas, **Habilitar finalización de pedidos** se debe establecer en **Sí**.</span><span class="sxs-lookup"><span data-stu-id="a3d30-109">When the Call Center channel is defined, **Enable Order Completion** must be set to **Yes**.</span></span> <span data-ttu-id="a3d30-110">Cuando se habilita la finalización de pedidos, los usuarios pueden ver el resumen de pedidos y hacer clic en **Enviar** para finalizar el pedido.</span><span class="sxs-lookup"><span data-stu-id="a3d30-110">When order completion is enabled, users can view the order recap and click **Submit** to finalize the order.</span></span> <span data-ttu-id="a3d30-111">Los usuarios también tendrán opciones para poner en espera manualmente el pedido de ventas en la revisión de fraudes.</span><span class="sxs-lookup"><span data-stu-id="a3d30-111">Users will also be provided options to manually place the sales order on fraud review hold.</span></span> <span data-ttu-id="a3d30-112">Los pedidos de ventas introducidos por un usuario del centro de llamadas se procesan con reglas y criterios de comprobación de fraudes durante el proceso de envío.</span><span class="sxs-lookup"><span data-stu-id="a3d30-112">Sales orders keyed in by a Call Center user are processed through fraud check rules and criteria during the submit process.</span></span>

<span data-ttu-id="a3d30-113">Existen dos tipos de criterios de fraudes a los que el sistema hará referencia para comprobar si el pedido se debe mantener en espera para revisión de fraudes:</span><span class="sxs-lookup"><span data-stu-id="a3d30-113">There are two types of fraud criteria that the system will reference to check if the order should be held for fraud review:</span></span>

-   <span data-ttu-id="a3d30-114">Las **reglas estáticas** utilizan un valor específico, como un número de teléfono que se ha puesto en una lista negra o una dirección de correo electrónico que se ha marcado porque se sabe que se ha usado para transacciones fraudulentas pasadas.</span><span class="sxs-lookup"><span data-stu-id="a3d30-114">**Static rules** use a specific value, such as a phone number that has been blacklisted or an email address that has been flagged as being known as used for past fraudulent transactions.</span></span> <span data-ttu-id="a3d30-115">En la página **Datos de fraudes estáticos**, la información de fraudes se puede agregar manualmente o mediante importación de datos, con puntuaciones adjuntas a la información fraudulenta.</span><span class="sxs-lookup"><span data-stu-id="a3d30-115">On the **Static Fraud Data** page, fraud information can be added manually or through data import, with scores attached to the fraudulent information.</span></span> <span data-ttu-id="a3d30-116">Si está activada la comprobación de fraudes, cada pedido de ventas especificado se comparará con los datos estáticos.</span><span class="sxs-lookup"><span data-stu-id="a3d30-116">If fraud checking is turned on, every sales order entered will be compared to the static data.</span></span> <span data-ttu-id="a3d30-117">Si los datos se encuentran en la dirección de facturación o de entrega del cliente, o si los datos se encuentran en la dirección de entrega de la línea de ventas, se sumarán las puntuaciones de todas las coincidencias únicas.</span><span class="sxs-lookup"><span data-stu-id="a3d30-117">If the data is found in either the customer’s billing or delivery address, or if the data is found in the delivery address on the sales line, the scores of all unique matches will be summed.</span></span>  
-   <span data-ttu-id="a3d30-118">Las **reglas dinámicas** se compone de variables y condiciones definidas para esas variables.</span><span class="sxs-lookup"><span data-stu-id="a3d30-118">**Dynamic rules** are composed from variables and conditions defined for those variables.</span></span> <span data-ttu-id="a3d30-119">Con las reglas dinámicas, ningún otro criterio definido en las reglas estáticas puede comprobarse.</span><span class="sxs-lookup"><span data-stu-id="a3d30-119">With dynamic rules, other criteria not defined in the static rules can be checked.</span></span> <span data-ttu-id="a3d30-120">Instrucciones “AND/OR” más complejas se pueden usar para considerar varias condiciones para determinar si hay una coincidencia positiva con los criterios de la regla y el pedido de ventas que se envía.</span><span class="sxs-lookup"><span data-stu-id="a3d30-120">More complex “AND/OR” statements can be used to consider multiple conditions to determine if there is a positive match against the rule criteria and the sales order being submitted.</span></span> <span data-ttu-id="a3d30-121">Por ejemplo, si un usuario desea poner en espera de revisión de fraudes todos los pedidos para clientes relacionados con un valor de grupo de clientes específico y que pidieron un producto específico, las condiciones para validar al cliente y para validar productos serían definidas en la página **Reglas** con la condición "AND".</span><span class="sxs-lookup"><span data-stu-id="a3d30-121">For example, if a user wants to hold for fraud review all orders for customers that are tied to a specific customer group value and that ordered a specific product, the conditions to validate customer and validate products would be defined on the **Rules** page with an “AND” condition.</span></span> <span data-ttu-id="a3d30-122">El pedido se pondría en espera por fraude solo si se cumplieran ambas condiciones y el valor de puntuación asignado a la regla pusiera la puntuación de fraude total del pedido por encima de la puntación de fraude mínima que se define en los parámetros del centro de llamadas.</span><span class="sxs-lookup"><span data-stu-id="a3d30-122">The order would fall into fraud hold only if both conditions were true and if the score value assigned to the rule put the order’s total fraud score over the minimum fraud score as defined in Call Center parameters.</span></span>

<span data-ttu-id="a3d30-123">Un usuario del centro de llamadas también puede configurar manualmente un pedido en la cola de retención por fraude.</span><span class="sxs-lookup"><span data-stu-id="a3d30-123">A call center user may also manually place an order into the fraud hold queue.</span></span> <span data-ttu-id="a3d30-124">Si el usuario que introduce el pedido cree que el cliente que realiza el pedido puede ser sospechoso y desea que otra persona revise adicionalmente la validez del pedido antes de procesarlo, el usuario que introduce el pedido puede elegir la opción **Retención manual por fraude** del menú desplegable **Retenciones** en la página **Resumen de pedido de ventas** (ésta aparece después de que se llama a la función **Completar** pedido.)</span><span class="sxs-lookup"><span data-stu-id="a3d30-124">If the user entering the order believes the customer placing the order may be suspicious and wants someone else to further review the validity of the order before it is processed, the user entering the order can choose the **Manual Fraud Hold** option from the **Holds** dropdown menu on the **Sales Order Summary** page (this appears after the **Complete** order function is called).</span></span> <span data-ttu-id="a3d30-125">Se solicitará al usuario que escriba una nota para explicar con más detalle por qué cree que el pedido puede ser fraudulento para que el revisor tenga más contexto.</span><span class="sxs-lookup"><span data-stu-id="a3d30-125">The user will be prompted to enter a note to further explain why they feel the order may be fraudulent so that the reviewer has more context.</span></span>

<span data-ttu-id="a3d30-126">Todos los pedidos retenidos a través de retención de fraude manual o por cálculo sistemático de las puntuaciones de fraude aparecerán en la página **Retenciones de pedido** , donde el pedido se puede revisar y después cancelar o liberar a procesamiento.</span><span class="sxs-lookup"><span data-stu-id="a3d30-126">All orders held through manual fraud hold or by systematic calculation of fraud scores will appear on the **Order Holds** page, where the order can be reviewed and then either canceled or released to processing.</span></span>

> [!NOTE]
> <span data-ttu-id="a3d30-127">El uso de varias reglas o de reglas excesivamente complejas producirá un rendimiento deficiente del sistema cuando se envíen pedidos de ventas.</span><span class="sxs-lookup"><span data-stu-id="a3d30-127">Use of multiple rules or overly complex rules will result in poor system performance when sales orders are submitted.</span></span> <span data-ttu-id="a3d30-128">La característica de alerta por fraude no se ha optimizado para gestionar un volumen grande de entradas de datos estáticos de fraudes y muchas reglas activas.</span><span class="sxs-lookup"><span data-stu-id="a3d30-128">The fraud alert feature hasn't been optimized to handle a large volume of static fraud data entries and many active rules.</span></span> <span data-ttu-id="a3d30-129">Es importante recordar que cada regla es evaluada durante la función de envío de la entrada de pedidos de ventas del centro de llamadas.</span><span class="sxs-lookup"><span data-stu-id="a3d30-129">It’s important to remember that every rule is evaluated during the submit function of call center sales order entry.</span></span> <span data-ttu-id="a3d30-130">Las reglas se evalúan con la cabecera del pedido de ventas y todas las líneas de pedido.</span><span class="sxs-lookup"><span data-stu-id="a3d30-130">The rules are evaluated against the sales order header and all order lines.</span></span> <span data-ttu-id="a3d30-131">Cuanto más reglas haya y más complejas sean las instrucciones de las reglas, más tardarán en procesarse.</span><span class="sxs-lookup"><span data-stu-id="a3d30-131">The more rules there are and the more complex the rule statements are, the longer it will take to process.</span></span> <span data-ttu-id="a3d30-132">Si tiene un gran número de artículos de línea en el pedido, y un gran número de reglas activas y entradas de datos estáticos, este proceso sistemático de revisar y validar todos los datos y de calcular una puntuación de fraude puede tener un impacto severo en el rendimiento.</span><span class="sxs-lookup"><span data-stu-id="a3d30-132">If you have a large number of line items on your order, and a large number of active rules and static data entries, this systematic process of reviewing and validating all of the data and calculating a fraud score can have a severe performance impact.</span></span>  <span data-ttu-id="a3d30-133">Las organizaciones que usan esta característica deben probar y confirmar siempre que el tiempo de procesamiento de envío de pedidos es aceptable antes de aplicar cambios a las reglas o los criterios de fraudes estáticos en el entorno de producción.</span><span class="sxs-lookup"><span data-stu-id="a3d30-133">Organizations using this feature should always test and confirm that the order submission processing time is acceptable before applying any changes to rules or static fraud criteria into the production environment.</span></span>

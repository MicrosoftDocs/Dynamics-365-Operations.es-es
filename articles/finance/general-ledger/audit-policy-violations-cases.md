# <a name="audit-policy-violations-and-cases"></a><span data-ttu-id="98bfb-104">Infracciones y casos de directivas de auditoría</span><span class="sxs-lookup"><span data-stu-id="98bfb-104">Audit policy violations and cases</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="98bfb-105">El artículo explica cómo se generan los casos de auditoría a partir de infracciones de reglas de directivas de auditoría.</span><span class="sxs-lookup"><span data-stu-id="98bfb-105">The article explains how audit cases are generated from violations of audit policy rules.</span></span> <span data-ttu-id="98bfb-106">También incluye información acerca de las distintas maneras en que las directivas de auditoría usan el intervalo de fechas de selección de documentos.</span><span class="sxs-lookup"><span data-stu-id="98bfb-106">It also includes information about the various ways that audit policies use the document selection date range.</span></span>

<a name="how-audit-cases-are-generated"></a><span data-ttu-id="98bfb-107">Cómo se crean casos de auditoría</span><span class="sxs-lookup"><span data-stu-id="98bfb-107">How audit cases are generated</span></span>
-----------------------------

<span data-ttu-id="98bfb-108">Las directivas de auditoría se usan para identificar los informes de gastos, los pedidos de compra y las facturas de proveedor que no cumplen con las reglas empresariales que defina y configure como reglas de directivas de auditoría.</span><span class="sxs-lookup"><span data-stu-id="98bfb-108">Audit policies are used to identify expense reports, purchase orders, and vendor invoices that don't comply with business rules that you define and configure as audit policy rules.</span></span> 

<span data-ttu-id="98bfb-109">Las directivas de auditoría se ejecutan en modo por lotes.</span><span class="sxs-lookup"><span data-stu-id="98bfb-109">Audit policies are run in batch mode.</span></span> <span data-ttu-id="98bfb-110">Cuando ejecuta una directiva de auditoría, todas las reglas de directivas que forman parte de esa directiva se ejecutan simultáneamente.</span><span class="sxs-lookup"><span data-stu-id="98bfb-110">When you run an audit policy, all the policy rules that are part of that policy are run at the same time.</span></span>

<span data-ttu-id="98bfb-111">Cada regla de directivas evalúa un conjunto de documentos.</span><span class="sxs-lookup"><span data-stu-id="98bfb-111">Each policy rule evaluates a set of documents.</span></span> <span data-ttu-id="98bfb-112">La regla de directivas selecciona documentos que se encuentran en el intervalo de fechas de selección del documento y coinciden con los criterios especificados.</span><span class="sxs-lookup"><span data-stu-id="98bfb-112">The policy rule selects documents that are in the document selection date range and that match the specified criteria.</span></span> <span data-ttu-id="98bfb-113">Por ejemplo, una regla de directivas puede seleccionar los informes de gastos con comidas que superen 50,00.</span><span class="sxs-lookup"><span data-stu-id="98bfb-113">For example, one policy rule might select expense reports that have meals that exceed 50.00.</span></span> <span data-ttu-id="98bfb-114">Otra regla de directivas puede seleccionar las facturas de proveedor que se deben pagar a un proveedor específico.</span><span class="sxs-lookup"><span data-stu-id="98bfb-114">Another policy rule might select vendor invoices that are payable to a specific vendor.</span></span> <span data-ttu-id="98bfb-115">Para cada documento que se seleccione del conjunto, se genera una infracción.</span><span class="sxs-lookup"><span data-stu-id="98bfb-115">For each document that is selected in the set, a violation is generated.</span></span> <span data-ttu-id="98bfb-116">Esa infracción es un registro de que un documento concreto, como la factura 12345, no cumple con la regla de directiva.</span><span class="sxs-lookup"><span data-stu-id="98bfb-116">That violation is a record that a particular document, such as invoice 12345, doesn't comply with the policy rule.</span></span> 

<span data-ttu-id="98bfb-117">Varios registros de infracción de auditoría se agrupan conjuntamente y se asocian a casos de auditoría.</span><span class="sxs-lookup"><span data-stu-id="98bfb-117">Multiple audit violation records are grouped together and associated with audit cases.</span></span> <span data-ttu-id="98bfb-118">De forma predeterminada, los casos para cada directiva de auditoría se agrupan por la regla de directivas de auditoría.</span><span class="sxs-lookup"><span data-stu-id="98bfb-118">By default, cases for each audit policy are grouped by audit policy rule.</span></span> <span data-ttu-id="98bfb-119">Si lo prefiere, puede seleccionar otros criterios de agrupación con la página **Criterios de agrupación de casos**.</span><span class="sxs-lookup"><span data-stu-id="98bfb-119">If you prefer, you can select other grouping criteria by using the **Case grouping criteria** page.</span></span> <span data-ttu-id="98bfb-120">Por ejemplo, puede agrupar encabezados de gastos por id. de proyecto y facturas de proveedor por cuenta de proveedor.</span><span class="sxs-lookup"><span data-stu-id="98bfb-120">For example, you can group expense headers by project ID and vendor invoices by vendor account.</span></span> <span data-ttu-id="98bfb-121">En este caso, todas las infracciones del encabezado de gastos con el mismo id. de proyecto se agruparán en el mismo caso, y todas las facturas de proveedor con la misma cuenta de proveedor se agruparán en el mismo caso.</span><span class="sxs-lookup"><span data-stu-id="98bfb-121">In this case, all expense header violations that have the same project ID will be grouped in the same case, and all vendor invoices that have the same vendor account will be grouped in the same case.</span></span> 

> [!NOTE]
> <span data-ttu-id="98bfb-122">Para las reglas de directivas de auditoría basadas en un tipo de consulta **Duplicado**, las infracciones no se agrupan por regla de directivas o por los criterios especificados en la página **Criterios de agrupación de casos**.</span><span class="sxs-lookup"><span data-stu-id="98bfb-122">For audit policy rules that are based on a **Duplicate** query type, violations aren't grouped by policy rule or by the criteria that are specified on the **Case grouping criteria** page.</span></span> <span data-ttu-id="98bfb-123">En su lugar, se agrupan por los criterios que se crean en la regla de directivas de auditoría.</span><span class="sxs-lookup"><span data-stu-id="98bfb-123">Instead, they are grouped by the criteria that are built into the audit policy rule.</span></span> <span data-ttu-id="98bfb-124">Por ejemplo, si una regla de directiva evalúa los informes de gastos para los gastos duplicados del mismo importe, id. de comerciante y fecha, todos los gastos que tengan los mismos valores en esos campos serán un caso.</span><span class="sxs-lookup"><span data-stu-id="98bfb-124">For example, if a policy rule evaluates expense reports for duplicate expenses of the same amount, merchant ID, and date, all expenses that have the same values in those fields will be one case.</span></span> <span data-ttu-id="98bfb-125">Los gastos que tengan valores diferentes, será un caso independiente.</span><span class="sxs-lookup"><span data-stu-id="98bfb-125">Any expenses that have different values will be a separate case.</span></span>

<span data-ttu-id="98bfb-126">Después de que se generen casos de auditoría, se gestionan con los procesos típicos para la gestión de casos.</span><span class="sxs-lookup"><span data-stu-id="98bfb-126">After the audit cases have been generated, they are handled by using the typical processes for case management.</span></span>

## <a name="document-selection-date-ranges"></a><span data-ttu-id="98bfb-127">Intervalos de fechas de selección de documentos</span><span class="sxs-lookup"><span data-stu-id="98bfb-127">Document selection date ranges</span></span>
<span data-ttu-id="98bfb-128">Cuando se ejecute una directiva de auditoría, cada regla de directivas selecciona documentos del tipo especificado que tengan una fecha que esté en el intervalo de fechas de selección de documentos.</span><span class="sxs-lookup"><span data-stu-id="98bfb-128">When an audit policy is run, each policy rule selects documents of the specified type that have a date that is in the document selection date range.</span></span> <span data-ttu-id="98bfb-129">El intervalo de fechas de selección de documentos se especifica en la página **Opciones adicionales**.</span><span class="sxs-lookup"><span data-stu-id="98bfb-129">The document selection date range is specified on the **Additional options** page.</span></span> <span data-ttu-id="98bfb-130">Muchos documentos tienen más de una fecha asociada.</span><span class="sxs-lookup"><span data-stu-id="98bfb-130">Many documents have more than one date associated with them.</span></span> <span data-ttu-id="98bfb-131">El campo de fecha que la regla de directivas de auditoría usa se especifica en la página **Tipo de regla de directivas**.</span><span class="sxs-lookup"><span data-stu-id="98bfb-131">The date field that the audit policy rule uses is specified on the **Policy rule type** page.</span></span>

<span data-ttu-id="98bfb-132">Estas son alguna otras maneras de que una directiva de auditoría use el intervalo de fechas de selección de documentos:</span><span class="sxs-lookup"><span data-stu-id="98bfb-132">Here are some other ways that an audit policy uses the document selection date range:</span></span>

-   <span data-ttu-id="98bfb-135">La directiva usa los nodos de la organización asociados con la directiva el último día del intervalo de fechas de selección de documentos.</span><span class="sxs-lookup"><span data-stu-id="98bfb-135">The policy uses the organization nodes that are associated with the policy on the last day of the document selection date range.</span></span> <span data-ttu-id="98bfb-136">Solo los nodos de la organización que están actualmente asociados a la directiva aparecen en la página de lista **Directivas de auditoría**.</span><span class="sxs-lookup"><span data-stu-id="98bfb-136">Only the organization nodes that are currently associated with the policy appear on the **Audit policies** list page.</span></span>
